Markdown の主要な記法をまとめますね。

見出し
# 見出し1
## 見出し2
### 見出し3
# の数で階層が決まります（1〜6まで）。

強調
**太字**
*斜体*
~~取り消し線~~
`インラインコード`
リスト
順序なしリスト：

- 項目1
- 項目2
  - ネスト項目
順序付きリスト：

1. 最初
2. 次
3. 最後
リンクと画像
[リンクテキスト](https://example.com)
![代替テキスト](画像URL)
コードブロック
``` で囲み、言語名を指定するとシンタックスハイライトされます：

```python
def hello():
    print("Hello")
```
引用
> 引用文
> 複数行も可能
水平線
---
表
| 列1 | 列2 |
|-----|-----|
| A   | B   |
| C   | D   |
:--- で左寄せ、:---: で中央、---: で右寄せ。

チェックボックス（GFM 拡張）
- [x] 完了したタスク
- [ ] 未完了のタスク
特に VRFSM の目論見.md などで使うことを想定するなら、コードブロックに言語指定をつける、表で状態遷移を整理する、> で設計上の注意点を引用ブロックにする、あたりが実用的です。

何か特定の用途（README、技術ドキュメント、ノート）向けの書き方があれば、もう少し踏み込んで説明します。
